Contributing to evalCalc

An introduction to contributing to the evalCalc project.

The evalCalc project welcomes, and depends, on contributions from developers and users in the open source community. Contributions can be made in a number of ways, a few examples are:

  • Code patches via pull requests
  • Documentation improvements
  • Bug reports and patch reviews
  • For information about available communication channels please refer to the README file in our GitHub repository.

Code of Conduct

Everyone interacting in the evalCalc project's codebases, issue trackers, chat rooms, and mailing lists is expected to follow the PyPA Code of Conduct.

Reporting an Issue

Please include as much detail as you can. Let us know your platform and evalCalc version. If the problem is visual (for example a theme or design issue) please add a screenshot and if you get an error please include the full error and traceback.
